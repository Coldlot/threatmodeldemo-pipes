# threatmodeldemo-pipes

Демонстрационный проект для отработки навыков **DevSecOps** и построения пайплайнов безопасности (SAST, DAST, SCA) для iOS-приложений.

Репозиторий содержит учебное iOS-приложение **ThreatModelDemo** с уязвимыми компонентами и полностью настроенный CI/CD pipeline в GitHub Actions.

---

## Функционал приложения ThreatModelDemo

| Компонент | Назначение |
|-----------|------------|
| **Jailbreak Detection** | Проверка, запущено ли приложение на взломанном устройстве (через `stat64`, `access`, `NSFileManager`, `fork`) |
| **SSL Pinning** | Закрепление сертификата для защиты от MITM-атак |
| **Keychain** | Сохранение и чтение токенов через Keychain API |
| **Hardcoded Secrets** | Учебные секреты (API-ключи, пароли) для проверки работы Semgrep и GitLeaks |
| **Network Requests** | HTTP/HTTPS-запросы к `httpbin.org` для проверки SSL Pinning |

## Настройки репозитория (Branch Protection)

| Правило | Значение |
|---------|----------|
| **Защищённые ветки** | `main`, `dev` |
| **Требование PR перед мержем** | ✅ Включено |
| **Требование прохождения статус-чеков** | ✅ Включено |
| **Обязательные чеки** | `Build IPA`, `Semgrep SAST`, `CodeQL Analysis`, `GitLeaks Secret Scan`, `MobSF Analysis` |
| **Запрет обхода правил** | ✅ Включено (даже для администраторов) |

**Результат:** Уязвимый код не может попасть в `main` или `dev` без прохождения всех проверок безопасности. Кнопка Merge заблокирована, пока все чеки не пройдут успешно.

## CI/CD Pipeline (`.github/workflows/ci.yml`)

Pipeline запускается автоматически при push в `main`/`dev` и при создании Pull Request в эти ветки.

### 1. Build IPA (`build`)

| Параметр | Описание |
|----------|----------|
| **Что делает** | Собирает iOS-приложение в `.ipa` файл без подписи |
| **Раннер** | `macos-latest` |
| **Результат** | Артефакт `threatmodeldemo-ipa` (`.ipa` файл) |
| **Зависимости** | Нет (выполняется параллельно) |

### 2. Semgrep SAST (`semgrep`)

| Параметр | Описание |
|----------|----------|
| **Что делает** | Статический анализ исходного кода (SAST) |
| **Что ищет** | Хардкод-секреты (`p/secrets`), уязвимости Swift (`p/swift`), OWASP Top 10 (`p/owasp-top-ten`) |
| **Раннер** | `ubuntu-latest` (Docker-контейнер `returntocorp/semgrep`) |
| **Результат** | SARIF-отчёт в артефакте `semgrep-results` и в GitHub Security Dashboard |
| **Блокирует PR** | ✅ Да (красный крест при находках) |

### 3. CodeQL Analysis (`codeql`)

| Параметр | Описание |
|----------|----------|
| **Что делает** | Глубокий анализ потока данных (Data Flow Analysis) |
| **Что ищет** | Логические уязвимости (SQLi, XSS, Path Traversal) в Swift-коде |
| **Раннер** | `macos-latest` |
| **Результат** | SARIF-отчёт в артефакте `codeql-results` и в GitHub Security Dashboard |
| **Блокирует PR** | ✅ Да (красный крест при находках) |

### 4. GitLeaks Secret Scan (`gitleaks`)

| Параметр | Описание |
|----------|----------|
| **Что делает** | Сканирует всю Git-историю на наличие секретов |
| **Что ищет** | API-ключи, токены, пароли, приватные ключи |
| **Раннер** | `ubuntu-latest` |
| **Результат** | SARIF-отчёт в артефакте `gitleaks-results`, бот оставляет комментарий в PR |
| **Блокирует PR** | ✅ Да (красный крест при находках) |

### 5. MobSF Analysis (`mobsf`)

| Параметр | Описание |
|----------|----------|
| **Что делает** | Статический анализ собранного `.ipa` файла |
| **Что ищет** | Хардкод-секреты, URL, сертификаты, небезопасные API, трекеры |
| **Раннер** | `ubuntu-latest` (Docker-контейнер `opensecurity/mobile-security-framework-mobsf`) |
| **Результат** | PDF-отчёт в артефакте `mobsf-report-pdf` |
| **Зависит от** | `Build IPA` (ждёт готовый `.ipa`) |
| **Блокирует PR** | ✅ Да (красный крест при находках) |

### 6. GitGuardian (автоматический)

| Параметр | Описание |
|----------|----------|
| **Что делает** | Проверяет каждый Pull Request на наличие секретов |
| **Что ищет** | API-ключи, токены, пароли, приватные ключи |
| **Результат** | Бот GitGuardian оставляет комментарий в PR с деталями находок |

### 7. Dependabot (автоматический)

| Параметр | Описание |
|----------|----------|
| **Что делает** | Мониторит зависимости на наличие известных уязвимостей (CVE) |
| **Что проверяет** | `Gemfile` (Ruby-зависимости, включая CocoaPods) |
| **Результат** | Автоматически создаёт PR с обновлением уязвимых зависимостей |

## Как использовать

1.  Клонировать репозиторий.
2.  Открыть `ThreatModelDemo.xcodeproj` в Xcode 16+.
3.  Запустить на симуляторе iOS 15.6+ (`⌘R`).

Для проверки CI/CD — создать Pull Request из любой ветки в `main` или `dev`. Pipeline запустится автоматически, и кнопка Merge будет заблокирована до успешного прохождения всех чеков.
